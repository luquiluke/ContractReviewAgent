# Scrutiny

## What This Is

Scrutiny is a contract review demo tool built for agency owners and small business owners. A user uploads a PDF contract, selects the contract type, and receives a plain-English section-by-section summary with basic PII stripped before the text reaches the AI. The output can be exported as a PDF or Excel report.

## Core Value

A prospect sitting across from you can upload any contract and walk away with a readable report in under 2 minutes — making them say "I need this for my business."

### Active

- [ ] English UI, jurisdiction-neutral (no country-specific references or assumptions)
- [ ] 8 contract types selectable manually: Employment, Freelance/Service, NDA, Vendor/SaaS, Letter of Assignment, Partnership, Lease/Rental, Loan/Financing
- [ ] Per-section plain-English summaries generated by LLM (2–4 sentences per section, ~4–5 sections per contract type)
- [ ] Basic PII stripping before text is sent to AI (PERSON, EMAIL, PHONE, IBAN, DATE — 5 types, demo-level)
- [ ] PDF export of the review report
- [ ] Excel export of the review report
- [ ] Scrutiny branding (name, dark theme, professional look)
- [ ] API key input via sidebar (no stored credentials)

### Out of Scope

- Automatic contract type detection — proven unreliable; manual selection is clearer for demos
- Risk assessment / risk levels — adds complexity without demo clarity; summary only
- Jurisdiction-specific legal analysis — this is a demo, not a legal product
- User accounts / authentication — not needed for demo use
- Production-grade PII coverage — basic 5-type stripping is enough to demonstrate the concept

## Context

- Rebuilt from the original `app.py` (Spanish, Argentina-focused, single-file Streamlit)
- Previous Phase 1–3 implementation abandoned due to complexity and UI bugs
- Stack: Python, Streamlit, FastAPI, LangChain + OpenAI (gpt-4o-mini), Presidio, Docker
- Existing infrastructure: `app/` module structure, Dockerfile, `scrutiny_theme.css`
- The app is a demo tool — polished appearance and reliable output matter more than edge-case coverage

## Constraints

- **Tech stack**: Python/Streamlit/LangChain/OpenAI — no rewrites, extend what exists
- **Scope**: Demo quality, not production-hardened — avoid over-engineering
- **PII**: Presidio-based, 5 entity types only — Presidio requires Docker (incompatible with Python 3.14 locally)
- **Model**: gpt-4o-mini — cost-effective for demo use
